# Code of Conduct

## Our Standard

Anchor is a small project with open-source ambitions. For now, contributions come
from a trusted circle — but the expectations here are written to scale as the
project grows.

Be direct, be professional, and keep the focus on the work.

**Expected behavior:**
- Communicate clearly and honestly
- Accept and give feedback on code and ideas without taking it personally
- Respect that the maintainer has final say on direction and decisions
- Keep discussions relevant to the project

**Unacceptable behavior:**
- Don't be an asshole!
- Personal attacks or sustained hostility toward contributors or maintainers
- Harassment of any kind, in any project space (issues, PRs, discussions)
- Deliberately introducing harmful or malicious code

## Scope

This Code of Conduct applies within all project spaces: GitHub issues, pull
requests, discussions, and any direct communication related to Anchor.

## Enforcement

Erik maintains this project and handles all conduct concerns. Reports will 
be handled with discretion. Consequences range from a private conversation 
to removal from the project,depending on severity.

## Attribution

This document may evolve as the project grows. Substantive changes will be
noted in the commit history.
